first steps

1. install

opam install js_of_ocaml-compiler
git clone git@github.com:maltasea/lua_of_ocaml.git
cd lua_of_ocaml
dune build

2. hello world

echo 'let () = print_endline "hello from lua"' > hello.ml
make hello
hello from lua

3. your own programs

edit hello.ml, make hello. compiles to bytecode, translates to lua, cleans up, runs.

any bytecode

./loo.sh prog.ml              # .ml → stdout
./loo.sh prog.ml out.lua      # .ml → out.lua
lua out.lua

./loo.sh is built by make. no install needed.

how it works

ocaml source
 → ocamlc → bytecode
  → jsoo parser (reused)
    → IR (Code.program)
      → generate_lua.ml → Lua AST
        → output_lua.ml → .lua text
          + runtime/lua/*.lua
          → luajit hello.lua

reuses jsoo's IR and optimizations. the code generator mirrors jsoo's generate.ml.

ffi: ocaml talking to lua

external becomes a direct lua call. a dummy c stub makes ocamlc happy. the compiler throws the stub away.

(* mylib.ml *)
external lua_add : int -> int -> int = "lua_add"
/* mylib_stubs.c — empty, just so ocamlc links */
#include <caml/mlvalues.h>
CAMLprim value lua_add(value a, value b) { return Val_int(0); }
ocamlc -c mylib_stubs.c
ocamlc -c mylib.ml
ocamlc -custom -o mylib.byte mylib_stubs.o mylib.cmo
./loo.sh mylib.byte -o mylib.lua

ocaml values in lua: ints tagged (n*2), floats boxed ({253, v}), strings plain, blocks {tag, f1, f2, ...}.
