🚀 First Steps

1. Install

opam install js_of_ocaml-compiler
git clone git@github.com:maltasea/lua_of_ocaml.git
cd lua_of_ocaml
dune build

2. Run hello world

echo 'let () = print_endline "hello from lua"' > hello.ml
make hello
hello from lua

3. Write your own

Edit hello.ml, then make hello. The Makefile compiles to bytecode, runs lua_of_ocaml, deletes intermediate .byte/.cmi/.cmo, and runs the generated hello.lua.

Any OCaml bytecode

./loo.sh prog.ml              # .ml -> stdout
./loo.sh prog.ml out.lua      # .ml -> out.lua
lua out.lua

./loo.sh is generated by make — zero install needed.

🔧 Architecture

OCaml source → ocamlc → bytecode (.byte)
  → js_of_ocaml parser (reused)
    → IR (Code.program)
      → lua_of_ocaml code generator (generate_lua.ml)
        → Lua AST
          → output_lua.ml
            → .lua text + runtime/lua/*.lua
              → lua hello.lua

The IR from js_of_ocaml is target-agnostic. lua_of_ocaml reuses the bytecode parser and optimizations, providing a structural CFG code generator modeled on js_of_ocaml's generate.ml.

🤝 FFI: Calling Lua from OCaml

OCaml external declarations become direct Lua function calls. A dummy C stub is needed only so ocamlc can link:

(* mylib.ml *)
external lua_fn : int -> int -> int = "lua_fn"
/* mylib_stubs.c — empty, just for linking */
#include <caml/mlvalues.h>
CAMLprim value lua_fn(value a, value b) { (void)a; (void)b; return Val_int(0); }
ocamlc -c mylib_stubs.c
ocamlc -c mylib.ml
ocamlc -custom -o mylib.byte mylib_stubs.o mylib.cmo
./loo.sh mylib.byte -o mylib.lua

The generated Lua calls lua_fn(...) directly. The C code is only compiled and linked — it is never executed. Same mechanism as js_of_ocaml's libjs_of_ocaml_stubs.a.

Defining the Lua side

Add functions to runtime/lua/misc.lua (or any .lua file loaded before the generated code). They become callable from OCaml via external:

-- in runtime/lua/misc.lua:
function lua_fn(a, b)
  -- a, b are tagged OCaml ints (value * 2)
  return a + b
end

OCaml values in Lua: integers are tagged (n*2), floats are boxed ({253, value}), strings are plain Lua strings, blocks are {tag, field1, field2, ...}.

Example: example-game/ — a LÖVE2D chicken jump platformer using OCaml → Lua FFI.

📝 Source Tracing

Generated Lua includes --# file:line comments at closure boundaries and block entry points mapping back to OCaml source. Compile with -g:

ocamlc -g -o hello.byte hello.ml
