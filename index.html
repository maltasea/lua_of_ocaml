first steps

1. install

opam install js_of_ocaml-compiler
git clone git@github.com:maltasea/lua_of_ocaml.git
cd lua_of_ocaml
dune build

2. hello

echo 'let () = print_endline "hello from lua"' > hello.ml
make hello
hello from lua

3. your own

edit hello.ml, make hello. compiles to bytecode, runs lua_of_ocaml, cleans up intermediates, runs the lua.

any bytecode

./loo.sh prog.ml              # .ml → stdout
./loo.sh prog.ml out.lua      # .ml → out.lua
lua out.lua

./loo.sh is generated by make. no install needed.

how it works

ocaml source
 → ocamlc → bytecode
  → js_of_ocaml parser (reused)
    → IR (Code.program)
      → generate_lua.ml → Lua AST
        → output_lua.ml → .lua text
          + runtime/lua/*.lua
          → luajit hello.lua

reuses jsoo's IR and optimizations. structural CFG codegen modeled on jsoo's generate.ml.

ffi: ocaml ↔ lua

external declarations become direct lua calls. dummy c stubs satisfy ocamlc's linker. the compiler replaces them.

(* mylib.ml *)
external lua_add : int -> int -> int = "lua_add"
/* mylib_stubs.c — empty, just for linking */
#include <caml/mlvalues.h>
CAMLprim value lua_add(value a, value b) { return Val_int(0); }
ocamlc -c mylib_stubs.c
ocamlc -c mylib.ml
ocamlc -custom -o mylib.byte mylib_stubs.o mylib.cmo
./loo.sh mylib.byte -o mylib.lua

ocaml values in lua: ints tagged (n*2), floats boxed ({253, v}), strings plain, blocks {tag, f1, f2, ...}.
